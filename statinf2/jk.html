Jackknife

Introduction

The jackknife is, as we have already seen, very much like CV where the test set is of size 1 and the training set is of size n-1. Thus, there are only n ways of partitioning a sample of size n into test and training set. As a result jackknife does not use any random sampling, rather averages over all the n possible partitions. Jackknife was originally introduced to estimate bias and variance of an estimator.

Jackknife estimate of bias

Set up: We have iid data X1,..., Xn. We have a statistic, T. It is a function that takes the sample as its input and produces a number as its output. We want to estimate bias(T).

Jackknife procedure: Call the entire sample as X. Define the i-deleted sample, X-i as

X-i = { X1,..., Xi-1, Xi+1,..., Xn}
This is our training set when the test set is {Xi}. Now compute
T-i = T(X-i)
and
T(.) = sum(T(X-i))/n
Then the jackknife bias estimator is defined as
biasjack(T) = (n-1)*(T(.) - T(X))

Example: Suppose we have data X = {2,5,6,3}. Thus n=4 here. T is the sample mean.

X-1 = {5,6,3} T-1 = 4.67
X-2 = {2,6,3} T-2 = 3.67
X-3 = {2,5,3} T-3 = 3.33
X-4 = {2,5,6} T-4 = 4.33
So T(.) = (4.67 + 3.67 + 3.33 + 4.33)/4 = 4.
Also, T(X) = (2 + 5 + 6 + 3)/4 = 4.
Hence biasjack(T) = (4-1)*(T(.) - T(X)) = 0.

Jackknife estimate of variance

Using same set up and notation as above
varjack(T) = sum(T-i - T(.))2)*(n-1)/n

Using the computer

Jackknifing is easy with S-plus. First we show the simple but crude way. Assume that the data is stored in an array called x. You already have a function T to compute the statistic.

S-plus allows negative index in an array to delete elements as follows. If x=(-9,2.5,4.6,5) then x[-3] = (-9,2.5,5). This makes it easy to create the deleted samples. 


> n <- length(x)
> for(i in 1:n)
+  val[i] <- T(x[-i])

> biasJack <- (n-1)*(mean(val)-T(x))

However, S-plus also has a ready-made function called jackknife. To use it you need a function to compute the statistic of interest. Let's call this function as T. Let the original data be stored in an array x. Then invoke the jackknife function as 


> val <- jackknife(x,T)
Then the variable val will store many important quantities. Type 

> names(val)
to get a list of these. Also see the help of jackknife for more information.

Justification of the jackknife estimator of variance

Jackknife, like any other resampling method, tries to "simulate" many datasets based on only one dataset (the original one). To understand any resampling method, therefore, it is best to start with the situation where we really can have many independent datasets. If we have many datasets and we want to estimate se of a statistic T, then we shall just compute T for each of the N datasets to get N numbers, say, T1,...,Tn. Then we can just compute the std dev of these as
varHat(T) = sum(Ti - Tbar)2/n,
where Tbar is the mean of the Ti's. Now compare this with
varjack(T) = sum(T-i - T(.))2*(n-1)/n
You can see that in the jackknife formula X-i's are playing the role of the fresh datasets. So T-i's are used in place of Ti's, and T(.) in place of Tbar.

The only thing that is still not explained is the choice of the constant. Why are we using (n-1)/n and not 1/n or 1/(n-1)? The answer lies in the following exercise.

Exercise JK.1: Suppose we want to estimate the variance of a statistic T by the following formula:
const*sum(T-i - T(.))2
and we want to choose const in a way that this estimator is unbiased if T is sample mean. Then show that const must be (n-1)/n.

Justification of the jackknife estimator of bias

A very similar interpretation exists for the jackknife estimate of bias. The constant is chosen to make the bias match the exact value for sample variance.

However, note one curious point. If T(X) is a constant function, then the jackknife estimator of its bias is zero! This is natural since in jackknife estimator of bias uses T(X) as a proxy for theta. So if this proxy is bad, the estimator is bad, as well. However, if T is "good" then this estimator is also "good". The following result makes this precise.

Theorem: If
E(T(X) = theta + b/n + O(1/n2),
then
E(Jackknife estimator of bias) = b/n + O(1/n2),
and hence
E(Jackknife bias-corrected T) = theta + O(1/n2).

Exercise JK.2: Prove the above theorem.

Thus, if T is already first-order unbiased (ie, bias term being b/n which is O(1/n)) then jackknife bias-corrected T is second-order unbiased (ie, bias term is O(1/n2)).

Which is better, jackknife or bootstrap?

Jackknife is certainly easier to compute, but it uses less information. It may be viewed as an approximation to bootstrap. We shall explore this in an assignment.

The main advantage of the bootstrap is its generality. One can do bootstrapping in a wide variety of set up. But it is quite difficult to generalise the notion of jackknifing beyond estimation of bias and variance. In fact, with bootstrap available, one would hardly consider doing jackknife.

Jackknife fails if the statistic is not "smooth". The median, for instance, is not smooth. Jackknife estimate of its std err is not consistent. However, the bootstrap estimate is.

Exercise JK.3: Suppose that we have iid sample X1,...,Xn. Let T(.) denote the sample median. Then how many distinct values can T-i's take as i varies? The original sample is held fixed.

Jackknife after bootstrap

Set up: We have iid data X1,...,Xn. We also have a statistic T. We have applied bootstrap to estimate some parameter, theta, of the sampling distribution of T. As is usual with bootstrap, consider theta as a function of the sampling distribution (e.g., mean or variance or 5% upper quantile). This estimator is itself a random variable. We may be interested in knowing how accurate it is. In particular we may want to estimate its variance. We may use jackknifing to do it.

However, this would require us to delete one point at a time and carry out the entire bootstrap process every time, which is very computation intensive. Jackknife-after-bootstrap is a clever method to reduce the computational load.

Suppose we draw N bootstrap resamples. Each bootstrap resample may be considered as drawing a set of n indices with replacement from {1,...,n}. Let the set Bk denote this set for the k-th resample.

Example: Suppose that our original sample is
X1, X2, X3, X4, X5, X6
Now let us draw a resample which consists of
{X3, X6, X6, X1, X2, X3}
If this is the k-th resample then
Bk = {3,6,1,2}

Next define Ci as follows for i=1,...,n.

Ci = { k : Bk does not contain i}

Example: (contd) Suppose we draw 5 resamples with the following Bk sets: 
   
   B1={3,6,1,2}

   B2={2,5,4,3}

   B3={2,1}

   B4={3,6,2,4,1,5}

   B5={4,6,2}
Then
C1={2,5}, since B2 and B5 does not contain 1.
Assume Ci's are all nonempty. Let Tk denote the T-statistic computed from the k-th resample. For each i consider the Tk's such that k is in Ci. These do not involve Xi. Use the empirical distribution of these Tk's as a proxy for the sampling distribution of T. Compute theta of this empirical distribution, call it thetai.

Example: (contd) Suppose that we take theta = variance. Let's compute theta1 in the above example. For this we shall need T2 and T5. Let's say these are 3 and 29, respectively. So
theta1 = (32 + 292)/2 - ((3+29)/2)2.

Let

thetaBar = mean(thetai).
Then the jackknife-after-bootstrap estimate of variance is defined as
Varjack(theta) = [(n-1)/n] * sum((thetai - thetaBar)2)

Fortunately, there is a S-plus function to do all these for you. Find it out!
