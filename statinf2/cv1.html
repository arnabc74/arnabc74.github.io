Cross-validation I

Cross-validation concept

When we model some random scenario our aim is to have predictions as close as possible to the data that will come in the future. However, this validation can only be done when the future data arrive, which may be too late. So we often use the already available data as a proxy for future data, and look at the residuals that result from the very data based on which we have fit our model. This procedure runs the risk of "overfitting" the data, ie the residual from the present data may be small but that for future data may be large.

Cross-validation is a way to counter overfitting. The main idea is as follows.

  1. Split the available data into two parts, called training and test data.
  2. Perform the modelling using only the training data.
  3. Judge the goodness-of-fit of the fitted model using only the test data.
  4. Repeat the above three steps a number of times using different (training,test) partitions, and average the goodness-of-fit.
The above algorithm will produce for a given data and given modelling strategy a single number called the cross-validation score. This score measures the good-of-fit of this modelling strategy for this data.

Models and Modelling strategies

A modelling strategy is like a computer program that takes as its input a dataset, and produces a fitted model. Be careful to distinguish between a model and a modelling strategy. A modelling strategy is a function that maps a dataset to a model. The model-fitting is just another name for the evaluating this function.

Example: "Least square fitting of a quadratic equation" is a modelling strategy that may result in the model
yi = 1.2 - 5.64*xi + 2.3*xi2 + epsiloni, where epsiloni ~ N(0,0.342)
for some given data set. The same model could also be obtained from a "least absolute deviation fitting of a quadratic equation", which is a different modelling strategy.

CV to choose between modelling strategies

Suppose we have k different modelling strategies:
{MS1,...,MSk}
Let M denote the set of all possible models, and let DAT denote the set of all possible datasets. So each MSi is a map from DAT to MOD.
MSi : DAT --> MOD
For each model M in MOD and D in DAT we have a way to measure how well the model M fits the data D. This is captured by a loss function
L : DAT x MOD ---> R,
where R is the set of reals.

Example: In simple linear regression with one regressor variable x and one response variable y a typical dataset consists of two columns of numbers, one giving values of x, the other giving the corresponding values of y. So here DAT consists of all possible pairs of columns. Suppose that MOD consists of all linear equations of the form
y = alpha + beta * x + epsilon,
where epsilons are iid N(0,sigma2). A popular modelling strategy here is the mle, which is just the least squares method. So given a dataset this modelling strategy will produce alphaHat, betaHat and sigma2Hat. The loss function L may be taken to be the squared error loss, ie,
L(X,M) = sum of squares of (y-yHat)'s,
where
yHat = alphaHat + betaHat*x

We want to know which modelling strategy is the best based on a given dataset X in DAT. Each strategy produces a model:

Mi = MSi(X).
One may now think of three ways to compare the modelling strategies:
  • Ideal method
  • Resubstitution method
  • Crossvalidation

Ideal method

Here we collect more data Y afresh after finding Mi based on the already available data X. Then we test how well Mi performs on Y by considering
L(Y,Mi).
This is, as we have already said, the ideal way of doing things, and should be used whenever possible. However, often collecting fresh data Y is prohibitively expensive. Then the ideal method is not applicable.

Resubstitution method

This is a naive way to compare the modelling strategies when fresh data are not available. Here we just test Mi on the original data X, ie we look at
L(X,Mi).

This is not a good way to compare modelling strategies because of the risk of overfitting. Since Mi was fitted based on X, it is likely that L(X,Mi) will be smaller than L(Y,Mi) where Y is some fresh data. If L(X,Mi) is much smaller than L(Y,Mi) then the loss is underestimated, and we have a case of overfitting.

Crossvalidation

This is a balance between the two methods above. It avoids overfitting, yet does not require fresh data. Here we look at the CV score of each MSi, and choose the strategy with the minimum CV score.

To do this we need some standard way of doing the (training,test) partition of the available data. There are two popular techniques:

  • p-fold CV (more popular)
  • leave-one-out CV (less popular)

p-fold CV

Here we first split the data randomly into p equal parts (or almost equal parts if the sample size is not divisible by p). Let the parts be called X1,...,Xp. Use Xt as the test set, and the rest of X as the training set. Let us denote by X-t this remaining part of X. Then we apply the modelling strategies on X-t to get
Mit = MSi(X-t)
Apply this to the test set, ie, Xt to get the loss
L(Xt,Mit)
Do this as t ranges from 1 to p, and then average to get the CV score for MSi:
CVi = sumt(L(Xt,Mit))/p
Now choose the modelling strategy that minimises this. However, observe that CVi's are themselves random quantities. So minor differences between them may very well be due to chance rather than any real performance difference among the corresponding modelling strategies.

leave-one-out CV

This is just like p-fold CV with p=n, the sample size. Thus, here we leave one datapoint out, fit a model and check the performance of the model on that left-out datapoint. We do it separately for each of the n datapoints, and then average. This has the advantage that it involves no randomness, so may sometimes be compute analytically. But a test data of size 1 is often judged inadequate.
